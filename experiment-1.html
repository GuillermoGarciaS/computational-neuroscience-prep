Experiment #1 — File Size, Structure, and Predictability

A small experiment about how controlled structure becomes measurable output. The point is not that file size equals meaning; it is that predictable structure can be modeled before deeper claims are made.

Introduction

Central question

How does increasing text length affect file size, and what can file size reveal about structure or predictability?

Boundary

This models storage growth, not semantic value or experience.

Variables

Independent: page count Dependent: file size in bytes Controlled: format, layout, margins, repeated content

Method

The method is compressed into a protocol map so the reader can understand the experiment without scrolling through six large cards.

1

Generate files

Create files with increasing page counts.

2

Control structure

Use repetitive content to keep the text pattern stable.

3

Measure bytes

Record the file size for each generated file.

4

Compare changes

Inspect whether consecutive size differences are regular.

5

Model growth

Estimate file size as a linear function of page count.

6

Limit the claim

Discuss predictability without confusing it with meaning.

Interpretation

The model is showing a small file with relatively low storage growth.

Core idea

Under repetitive and controlled input, file growth becomes regular and easy to model. That makes the relation between page count and file size predictable, but it still does not make file size a direct measure of meaningful information.

Reflection

What this explains

The experiment explains why controlled input can produce regular, predictable output. It is a first layer of measurement.

What it does not explain

It does not explain semantic meaning, interpretation, or subjective experience. Predictability is not consciousness.
