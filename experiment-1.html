Experiment 1 — File Size, Structure, and Predictability

Introduction

Central question

How does increasing text length affect file size, and what can file size reveal about structure or predictability?

Adaptation

From homework to experiment

This version keeps the statistical core of the assignment, but reframes it as an interactive study of structure, predictability, and measurement.

Open details ↓

The original homework asks for ten text documents with increasing page counts, measurement of file size in bytes, calculation of differences between files, and a linear regression of size as a function of number of pages. This longer detail is still available, but it no longer interrupts the first reading of the page.

Why it matters

This experiment offers a minimal case of how structured input can become measurable output. It does not explain semantics, perception, or subjective experience, but it does help map a first layer of the broader project: how computational systems transform input into quantifiable form.

Variables

Independent Variable

Number of pages in each file.

Dependent Variable

File size in bytes.

Controlled Variables

File format, font, layout, margin settings, and the type of repeated content used.

Method

Step 1 — Generate files

Create ten files with increasing page counts, from one page to ten pages.

Step 2 — Keep the structure simple

Use highly repetitive content so that the textual pattern remains controlled across files.

Step 3 — Measure storage

Record the file size in bytes for each file.

Step 4 — Compare differences

Compute differences between consecutive file sizes to inspect regularity.

Step 5 — Model the pattern

Apply linear regression to estimate file size as a function of page count.

Step 6 — Interpret carefully

Use the result to discuss predictability and formal structure, but not semantic meaning or subjective experience.

Current interpretation

The model is showing a small file with relatively low storage growth.

More pages and more letters per page increase total file size, while base overhead shifts the curve upward. This models storage growth, not semantic meaning.

Insight

Under repetitive and controlled input, file growth becomes regular and easier to model. This makes the relationship between page count and file size more predictable, but it does not mean file size directly measures meaningful information.

Reflection

A file filled with repeated characters may be easy to predict and regular in structure, yet still contain little semantic value. This suggests that storage, predictability, and meaning are related but not identical.

Computational systems can track regularity and reduce uncertainty in formal structures, but that alone does not explain interpretation or subjective experience.
