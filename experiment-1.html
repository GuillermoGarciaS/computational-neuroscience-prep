Experiment 1 — Entropy and Structure

Question

How does increasing structure in input data affect predictability and entropy in a computational system?

Setup

System

The experiment generates text files with increasing amounts of structured content.

Input Range

Files go from 1 to 10 pages, each filled with repeated characters.

Observation

Each file is measured in bytes to observe how size changes as a function of content.

Method

Step 1 — Generate structured files

Create 10 text files with increasing pages of repeated characters.

Step 2 — Measure output

Record file size in bytes for each file.

Step 3 — Compare change

Compute differences between consecutive files to examine stability.

Step 4 — Model relationship

Apply linear regression: Size = f(Pages).

Step 5 — Interpret predictability

Analyze whether structured input produces stable and predictable output.

Results

File size increases approximately linearly with the number of pages, indicating a highly predictable system under structured input.

This live graph simulates how a near-linear file-size relationship looks as the growth rate changes.

Differences between consecutive files remain relatively stable, showing low variability.

Insight

The system exhibits low entropy because the input is highly structured. As the number of pages increases, the output (file size) remains predictable and follows a consistent pattern.

This shows that structured systems reduce uncertainty, making outcomes easier to model and approximate.

Reflection

A file filled with repeated characters has very low entropy and is highly predictable, yet it carries almost no meaningful information. This suggests that low entropy alone does not imply meaningful structure.

Computational systems can measure predictability, but they do not interpret or experience the content they process. Entropy helps describe uncertainty in representation, but not meaning or subjective experience.
