Experiment #4 — Detectability of Structure

Experiment 3 showed that entropy depends on the observation window. This experiment asks a sharper question: can two signals have similar entropy while one has a clearer detectable pattern?

Introduction

Central question

When does a sequence contain a detectable structure, and why is entropy alone insufficient to answer that?

Why this matters

Structure is not only frequency

A sequence can contain the same number of 0s and 1s as another sequence, yet have a very different order. Entropy based on symbol frequency may treat them as similar, even when one has a visible pattern.

Theory

Shannon entropy measures uncertainty in a chosen representation. If the representation only counts how many 0s and 1s appear, then it can miss the order in which those values appear.

H(X) = - Σ p(x) log₂ p(x)

For a binary sequence, maximum entropy occurs when 0 and 1 are balanced. But balance is not the same thing as structure. The sequences 010101 and 001101 may both contain a mix of 0s and 1s, but the first has a strong alternating order.

A(S) = switches / (n - 1)

The alternation score measures how often the sequence changes from 0 to 1 or from 1 to 0. A perfect alternating sequence has a score close to 1. A constant sequence has a score of 0.

R(S) = average length of consecutive runs

Run length measures how long the same value tends to repeat before switching. Long runs suggest block-like structure. Very short runs suggest alternating structure. This gives us a second lens beyond entropy.

Detection = metric > threshold

A detector is a rule. It does not magically understand the sequence. It only checks whether a measurable property crosses a threshold. This is useful because it forces us to define structure operationally.

Examples

Alternating

0101010101

This sequence can have high entropy because 0s and 1s are balanced, but it also has strong order because it switches almost every step.

Balanced shuffled

0011010110

This can have similar entropy to an alternating sequence, but its order is weaker. This is the core reason entropy alone is not enough.

Constant

1111111111

This has very low entropy because the next symbol is predictable. But low entropy does not automatically mean rich structure.

Method

Variables

Independent: sequence type Independent: noise level Independent: threshold Dependent: entropy Dependent: alternation score

Procedure

Step Action Reason
1 Generate or enter a binary sequence. Keep the representation controlled.
2 Calculate entropy from the count of 0s and 1s. Measure uncertainty from symbol balance.
3 Calculate alternation and run-length scores. Measure order that entropy can miss.
4 Apply a threshold-based detector. Define pattern detection operationally.

Interpretation

The key result is that uncertainty and detectability are not the same thing. Entropy tells us how balanced or predictable the symbols are. A detector tells us whether a specific kind of structure is present according to a rule.

This matters for the larger project because representation cannot be reduced to lower entropy alone. A system must also preserve or extract useful relations among elements. In this experiment, that relation is simple: alternation. Later, it can become correlation, prediction, feature detection, or multi-channel integration.

Limits

Main limitation

This detector only finds one kind of structure: alternation. A sequence can fail the alternation detector while still containing another pattern, such as blocks, repetition every five steps, or a hidden rule. A detector always reflects the structure it was designed to find.

Core idea

Entropy measures uncertainty. Detection measures whether a chosen structure is identifiable.

Warning

A detector can only find what it is designed to detect.
