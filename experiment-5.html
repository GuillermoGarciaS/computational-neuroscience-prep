Experiment #5 — Representation Under Constraint

Experiment 4 showed that a rule can detect structure. This experiment asks whether the detected structure becomes useful for a task once the system has limited memory and a limited feature budget.

Introduction

Central question

When does a detected pattern become a usable representation?

Page thesis

Detection is not enough

Detection finds structure. Representation preserves structure in a form that can be used for classification, prediction, reconstruction, compression, or action.

Theory

A detector can notice alternation, blocks, runs, frequency, or transition rate. But a system still needs to keep some of those features and use them in a task. Under constraint, representation becomes selective preservation, not perfect copying.

Representation = retained useful features / available structure

The lab below uses four toy signal families. When the feature budget is small, the representation may only separate structured from random. When the budget expands, it can distinguish alternation from blocks or long runs.

Interpretation

The central result is not just whether a feature is detectable. The important result is whether the retained feature set lets the system do something better than guessing.

This keeps the project from treating representation as a magical label. Here, representation means a compressed internal state that preserves enough structure to support behavior.

Limits

Main limitation

This is still a toy representation. It uses simple hand-designed features and known labels. Real biological or machine systems learn features under many more pressures, including embodiment, reward, memory, and environmental feedback.

Core idea

Representation is useful structure under constraint.

Warning

A detected pattern is not yet a representation unless it helps a task.
